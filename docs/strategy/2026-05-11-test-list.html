F · cooking timers new in 16.6

Paprika-style inline tappable times + sticky timer bar + multi-timer + wakelock. See mockup at docs/strategy/2026-05-11-cooking-timer-mockup.html.

  • Open a recipe with cookable times (e.g. "Bake for 25 minutes"). Inline time text renders as a small terracotta pill. Steps without parseable times render plain — no layout break.
  • Tap an inline time pill → bottom-sheet picker opens with the matched duration pre-selected.
  • Tap Start <N>-minute timer → sheet dismisses, sticky terracotta timer bar appears at top of body.
  • Timer counts down VISIBLY at 1-second cadence — chip text updates each second (mm:ss). Was frozen on the previous APK; fixed in b7e1820.
  • Tap the running chip → pauses (mocha-grey, dot disappears, freeze).
  • Tap the paused chip → resumes from the frozen remaining time (not a fresh start).
  • Long-press a chip → confirm → "Cancel timer" removes it; "Keep" leaves running.
  • Multi-timer — start two on different steps → both visible at top.
  • Max-concurrent cap — start 5 → 6th attempt → snackbar "You have the maximum number of timers running."
  • Expiry alert (foreground) — haptic buzz + TTS speaks "Step N timer done" + snackbar with OK → OK removes chip.
  • Custom duration — tap "custom…" chip → Material time picker (24h) → pick → returns to picker → Start works.
  • Range "2–3 minutes" deliberately skipped — no pill rendered for ambiguous ranges. By design.
  • No regressions: servings adjust, ingredient swap, side dish gen, hands-free voice, share, bookmark, generate-another, feedback bar — all still work.

G · wakelock while timers active
