# Elio — AI Recipe Generator

> "Already knows your kitchen."

Elio is a Flutter app that generates personalised recipes using Google Gemini. It knows your pantry staples, dietary requirements, and household preferences — so you only need to tell it what's fresh today.

---

## Tech Stack

| Layer | Technology |
|---|---|
| Framework | Flutter 3.27.x (Dart 3.6.x) |
| AI | Google Gemini (gemini-flash-latest) |
| Auth | Firebase Auth + Google Sign-In |
| Database | Cloud Firestore |
| Local storage | shared_preferences |
| Fonts | Google Fonts (Outfit) |

## Getting Started (Android)

### Prerequisites
- Flutter 3.27+ (`flutter --version`)
- Android SDK with API 23+ and NDK 27.0.12077973 (`flutter doctor`)

### Setup
```bash
git clone https://github.com/robchambo/elio-app.git
cd elio-app
flutter pub get
```

### Firebase setup (required for Google Sign-In and Firestore)
1. Open the Elio project in [console.firebase.google.com](https://console.firebase.google.com)
2. Download `google-services.json` → place at `android/app/google-services.json`
3. Run `flutterfire configure` to regenerate `lib/firebase_options.dart`
4. Add your debug keystore SHA-1 to Firebase:
   ```bash
   keytool -list -v -keystore ~/.android/debug.keystore -alias androiddebugkey -storepass android -keypass android
   ```

### Build & install
```bash
flutter build apk --debug
adb install -r build/app/outputs/flutter-apk/app-debug.apk
```

## Getting Started (iOS)

> **Requires macOS with Xcode 15+ installed.**

### Prerequisites
- Xcode 15+ (Mac App Store)
- CocoaPods: `sudo gem install cocoapods`
- Apple Developer account (free for simulator; $99/year for device/TestFlight)
- Flutter 3.27+

### Setup
```bash
git clone https://github.com/robchambo/elio-app.git
cd elio-app
flutter pub get
cd ios && pod install && cd ..
```

### Firebase setup (iOS-specific)
1. In Firebase Console, add an iOS app with bundle ID `com.elio.elioApp`
2. Download `GoogleService-Info.plist` → drag into `ios/Runner/` **via Xcode** (not Finder) so it is added to the build target
3. In `ios/Runner/Info.plist`, replace `REVERSED_CLIENT_ID_PLACEHOLDER` with the `REVERSED_CLIENT_ID` value from `GoogleService-Info.plist`
4. Run `flutterfire configure` to regenerate `lib/firebase_options.dart`

### Run
```bash
# Simulator
flutter run

# Physical device (requires paid Apple Developer account + Xcode signing)
open ios/Runner.xcworkspace   # Set team in Signing & Capabilities, then run from Xcode
```

## App Icon Generation (Sprint 7)

When the final brand icon is ready:

1. Place a **1024×1024 PNG** at `assets/icon/app_icon.png`
2. Place a **1024×1024 foreground-only PNG** (transparent bg) at `assets/icon/app_icon_foreground.png`
3. Run: `flutter pub run flutter_launcher_icons`

This generates all Android and iOS icon sizes automatically. Config is in `flutter_launcher_icons.yaml`.

## Architecture

```
lib/
  main.dart                  # Entry point, auth gate, guest mode routing
  firebase_options.dart      # Generated by flutterfire (not committed)
  theme/
    elio_theme.dart          # Design system: colours, typography, ThemeData
  models/
    elio_models.dart         # Core domain models
    onboarding_state.dart    # Onboarding data model
    recipe_models.dart       # Recipe request/response/saved models
  services/
    auth_service.dart        # Firebase Auth + Google Sign-In
    firestore_service.dart   # Firestore read/write, daily cap
    gemini_service.dart      # Gemini API — recipe generation
    history_service.dart     # Local recipe history (shared_preferences)
  screens/
    onboarding/              # Screens 0–5 (welcome → style preferences)
    home/                    # Home screen with mood chips and perishables input
    recipe/                  # Recipe display, serving scaler, share
    history/                 # Local recipe history list
  widgets/
    elio_progress_bar.dart   # Shared onboarding progress indicator
```

## Environment Notes

- The Gemini API key is hardcoded in `lib/services/gemini_service.dart` for development. Sprint 4 will move this to a backend proxy.
- `lib/firebase_options.dart` is a placeholder — replace with `flutterfire configure` output before enabling Firebase.
- `google-services.json` and `GoogleService-Info.plist` are gitignored and must be added manually after cloning.
- Minimum Android SDK: 23. Minimum iOS: 16.0.
